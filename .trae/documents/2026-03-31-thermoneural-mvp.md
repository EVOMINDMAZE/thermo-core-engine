# Thermoneural MVP Implementation Plan

> **For Claude / Trae:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.
> Note: The user requested parallel execution via subagents, so tasks that don't block each other can be assigned to different subagents.

**Goal:** Build the MVP for Thermoneural: a Streamlit dashboard that ingests cryogenic sensor logs, detects anomalies using an Isolation Forest model, applies domain-specific rules (e.g., scroll valve leak), and generates a professional PDF RCA report using WeasyPrint. Also includes the Master Blueprint, Asset Plan, and Outreach Plan.

**Architecture:** A multi-layer Python application using the `src/` layout. It includes a synthetic data generator (PULSE library equivalent), an unsupervised learning model layer (Isolation Forest), a rule-based expert system layer for root-cause mapping, a reporting layer for generating PDF artifacts via WeasyPrint, and a Streamlit frontend for user interaction.

**Tech Stack:** Python 3.10+, Streamlit, pandas, numpy, scikit-learn, joblib, matplotlib/plotly, WeasyPrint, pytest. Dependency management via `pip` and `requirements.txt`.

## 1. Master Blueprint & Technical Spec

### Repository Structure
```text
Thermo-core/
├── requirements.txt
├── README.md
├── src/
│   ├── thermoneural/
│   │   ├── __init__.py
│   │   ├── data/
│   │   │   ├── __init__.py
│   │   │   └── synthetic.py      # Generates synthetic sensor logs
│   │   ├── models/
│   │   │   ├── __init__.py
│   │   │   └── anomaly.py        # Isolation Forest logic
│   │   ├── rules/
│   │   │   ├── __init__.py
│   │   │   └── expert_system.py  # Domain rules mapping anomalies to failures
│   │   └── reports/
│   │       ├── __init__.py
│   │       ├── generator.py      # WeasyPrint PDF generation
│   │       └── templates/
│   │           └── report.html   # HTML template for PDF
│   └── app.py                    # Streamlit Dashboard Entrypoint
└── tests/
    ├── __init__.py
    ├── test_synthetic.py
    ├── test_anomaly.py
    ├── test_expert_system.py
    └── test_reports.py
```

### Core Interfaces
- **Data Input:** CSV with `timestamp`, `temperature`, `pressure`, `vibration`, `current`.
- **Model Output:** DataFrame with added `is_anomaly` (boolean) and `anomaly_score` (float).
- **Rule Engine Output:** Dictionary containing `failure_mode` (e.g., "Scroll Valve Leak"), `confidence_score` (0-100%), `estimated_time_to_failure` (days), and `recommended_actions`.
- **Report Generator:** Takes the raw data, model output, and rule output to produce `report.pdf`.
